[19:54:36] Natalia Kravchenko: По поводу гит - смотри
схема сейчас проста(исходим из того, что у тебя уже склонирован проект + доступна гит конфигурация)
1) берешь переключаешься на мастер ветку: команда 
git checkout master
2) на всякий случай подтягиваешь изменения из удаленного репозитория с помощью команды
git pull origin master
3) допустим, что ты фиксишь 2-ой баг..как следствие - создаешь новую ветку из ветки мастер; выполняешь команду:
git checkout -b new_branch_name master
в твоем случае, можешь писать
git checkout -b bugfix/Oleg_bug_2
4) ты уже переключился на новую ветку, поэтому можешь сделать ряд изменений, которые необходимы для фикса и уже 
после команды по переносу изменений в стейджинг область - команда:
git add .
после уже коммит - фиксирование данных
git commit -m "fix for bug2 is available"
5) имея новую ветку, коммит (можешь проверить логи посредством команды
git log --all  --graph 
или
git log -n 2 (покажет 2 последних коммита)

6) можно пушить..при этом смотри, есть небольшой момент 
bugfix/Oleg_bug_2
ветка существует только у тебя локально 
для того, чтобы создать ветку еще удаленно нужно использовать опцию --set-upstream

то-есть выполнишь команду
git push --set-upstream origin  bugfix/Oleg_bug_2
